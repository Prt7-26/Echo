# Echo Desktop Shell

A thin Tauri 2 wrapper around the Hermes Echo dashboard that adds two
signal sources the browser sandbox cannot capture:

1. **OS clipboard** — polled every 2 s; when the text changes the shell
   POSTs a `clipboard_copy` signal to the Echo dashboard API. Only the
   **length** and the first **200 chars** are sent; raw clipboard
   contents are not persisted server-side.
2. **Window focus / blur** — fired into the same endpoint with
   `event_type=window_focus` / `window_blur`, so Echo can detect "user
   bailed out right after the agent replied".

Both events land in `echo_signal_event` (Layer A) attributed to the most
recent invocation. The full payload contract lives in
[plugins/echo_signals/dashboard/plugin_api.py](../plugins/echo_signals/dashboard/plugin_api.py)
under `POST /api/plugins/echo_signals/clipboard-signal`.

## Why Tauri?

The Echo proposal targets a desktop posture that the browser security
model cannot provide. Tauri gives us:

* Native OS clipboard read via `tauri-plugin-clipboard-manager`.
* Window-event hooks (`window_focus`, `window_blur`).
* A single Rust binary per OS (macOS / Windows / Linux) — no Electron
  bundle weight.
* Webview reuse: the dashboard React SPA runs as-is in the embedded
  webview pointed at `http://127.0.0.1:9119/echo`. No code duplication.

## Prerequisites

* Rust toolchain (`rustup` + stable channel) — https://rustup.rs
* Node.js (any LTS) for `@tauri-apps/cli`
* Platform-specific webview deps:
  * **macOS**: nothing — WKWebView ships with the OS.
  * **Linux**: `webkit2gtk` + GTK 3 dev packages
    (`sudo apt install libwebkit2gtk-4.1-dev` on Debian/Ubuntu).
  * **Windows**: WebView2 Evergreen runtime (usually pre-installed on Win 11).
* Hermes Echo dashboard running locally:
  ```bash
  hermes plugins enable echo_signals
  hermes dashboard
  ```
  The shell loads `http://127.0.0.1:9119/echo` from the running
  dashboard process. If it's not up the bootstrap page will retry
  every second.

## Build & run

```bash
cd tauri-shell
npm install                  # installs @tauri-apps/cli only
npm run dev                  # dev mode, attaches to the running dashboard
npm run build                # produces a signed/unsigned bundle in
                             # src-tauri/target/release/bundle/
```

Bundle outputs:

* macOS: `target/release/bundle/macos/Echo.app` + `.dmg`
* Windows: `target/release/bundle/msi/Echo_0.1.0_x64_en-US.msi`
* Linux: `target/release/bundle/appimage/echo-shell_0.1.0_amd64.AppImage`
  + `.deb` for Debian-family distros.

## Privacy posture

* Echo's backend stores **only the first 200 chars** of any clipboard
  text (`value_text` column in `echo_signal_event`), plus the integer
  character count (`value_int`). The rest is dropped on receipt.
* No remote endpoints are contacted — the shell talks to `127.0.0.1`
  only. Verify by inspecting `src/clipboard.rs` (the URL constant is
  hardcoded).
* The clipboard polling task can be disabled at build time by
  removing the `tokio::spawn(... watch_clipboard ...)` call in
  `src/lib.rs`. Window-focus events are wired separately and can be
  removed independently.

## What's not yet here

* **Hermes session-token gate**: Hermes serves protected endpoints
  behind a session token injected into the dashboard HTML. The
  `clipboard-signal` endpoint is currently NOT behind that gate, so
  the shell can hit it without scraping the token. If a future Hermes
  release enforces token auth across all `/api` paths, `clipboard.rs`
  will need to fetch `/` first, parse `window.__HERMES_SESSION_TOKEN__`
  out of the HTML, and pass it as `X-Hermes-Session-Token` on every
  POST. The Tauri shell can read the served HTML directly via reqwest,
  so this is a small follow-up.
* **App icon**: `src-tauri/icons/icon.png` is a placeholder. Drop a
  512×512 PNG (or the full Tauri icon set generated by
  `npm run tauri icon`) before producing a release bundle.
* **Auto-launch Hermes**: today the user is expected to start
  `hermes dashboard` separately. A future iteration could spawn it
  as a child process at shell startup.
